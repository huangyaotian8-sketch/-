03 选择角色

已经知道要给谁做卡，直接手动输入名字即可（跳过扫描、最省 token）；不确定有谁再用「扫描」浏览全书。同一角色因昵称没合并时，选中它们点「合并所选」。

04 生成角色卡

选好角色后生成角色卡。

user 设置（把「你」写进角色的世界）
在上方角色列表里可点选多个角色，会一次批量抽取、合并到同一份输出里。
提示：模型抽取难免有偏差，建议生成后对照原文核对一遍再用。

05 生成世界书

从全书提取世界观 / 势力 / 地点 / 设定 / 历史等条目，附推荐的酒馆条目设置。可下载成能直接导入酒馆的世界书 JSON，或照预览手动建条目。

user 设置（世界书里也把「你」写进去）
导入方式：酒馆 → 世界信息 → 导入，选下载的 .json 即可。核心世界观默认设为常驻（蓝灯），具体条目设为关键词触发（绿灯）。

06 文风蒸馏

抽样分析全书写作风格（视角 / 句式 / 用词 / 节奏 / 情感表达等），产出一份可直接粘贴给 AI 的「文风提示词」，让它照着写出原作风味。只做风格抽象，不摘抄原文。

自定义要求（可选，留空用默认）

07 打包导出

流程最后一步：把上面生成好的角色卡、世界书、文风打包成一张酒馆可直接导入的完整角色卡（世界书内嵌绑定，文风自动写入条目）。缺哪部分就只打包已有的。

导入酒馆：角色管理 → 导入角色，选这个 .json，角色卡 + 绑定的世界书 + 文风一并到位。世界书名称可在上面「生成世界书」那栏填。

08 自检

对照原文检查已生成的角色卡 / 世界书条目有没有 OOC、经历互串、与原文冲突的错误，给出问题日志和修正后的成品。先「扫描当前内容」，勾选要查的项，再「开始自检」（只查勾选的，省 token）。不含历史记录。

09 历史记录

每次生成的角色卡会自动存到这里（本机浏览器，最多 30 条）。点「查看」读回上方输出框。

10 报错自查

遇到报错先对照下面几条，多数能自己解决。

Load failed / 网络错误 / 请求发不出去
请求没到服务器，通常是浏览器拦截：① 若用的是 http 接口，别在 https 页面里调（混合内容会被拦）；② 接口需支持跨域（CORS），不支持的换一个；③ 本地接口请确认它确实在运行。与模型、参数无关，别在那上面找原因。
429 / rate limit / 限流
请求太频繁或配额用尽。等一两分钟再试；换限流更宽的模型（如 flash 系）；或检查你所用接口的配额。工具已内置自动重试。
HTTP 500 / do_request_failed / upstream error
服务端错误，通常是你所用中转的上游出了问题（渠道挂了、没该模型、欠费等）。重试几次；点「拉取模型」从它实际支持的列表里选一个；仍不行就换模型或换服务商。
只输出思考、没正文 / content 为 null
思考型模型把 token 花在思考上了。把 max_tokens 调大（思考型建议上万），或改用非思考型 / flash 模型。
解析失败 / 返回为空
模型没按要求只输出 JSON，或内容被安全过滤挡了（成人/暴力题材常见）。看日志里显示的模型实际返回：若是空，多半是被过滤，换限制更宽的模型或渠道；若是乱七八糟的文字，重试或换模型。
导入小说是乱码
编码不匹配。工具会自动尝试 UTF-8 与 GB18030；若仍乱码，用文本编辑器把文件另存为 UTF-8 再导入。
拉取模型失败
该接口可能没开模型列表端点（报 404），或跨域限制。手动填模型名即可。
提示版权 / copyright 相关拒绝
部分模型对处理长篇作品有版权限制，会拒绝。换一个在这方面较宽松的模型来跑小说提取即可；也可把分段调小。
